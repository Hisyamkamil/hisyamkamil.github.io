🔑 RDM Token 2.1 Extension

Version: 2.1.0
Author: PT Mitra Solusi Telematika
Compatible with: PILOT Extensions Runtime, Ext JS 7.7+

📋 Overview

The RDM Token 2.1 Extension provides comprehensive rental unit token lifecycle management with JWT-based security. It enables automatic and manual token activation, tracks token expiration, and integrates with RDM systems for unit control while maintaining strict security protocols.

✨ Key Features

🎯 Token Management

Complete token lifecycle management including creation, activation, renewal, and expiration tracking with real-time status updates.

🗺️ Location Monitoring

Interactive map interface showing real-time unit locations with GPS tracking, geofence validation, and movement history.

📊 Dashboard Analytics

Comprehensive metrics display including active tokens, expired units, pending approvals, and operational statistics.

🔐 JWT Security

Advanced JWT-based token security with device signature validation, offline token validation, and cryptographic protection.

📋 Contract Integration

Full contract management integration with automatic validation, expiration tracking, and customer data synchronization.

✅ Approval Workflow

Multi-level approval system for token requests with automated notifications and workflow management.

🏗️ Architecture

This extension follows PILOT Extension Pattern 1 - Full UI Extension with navigation tab and main content panel integration:

  • Navigation Tab: Left sidebar with multiple sections (Dashboard, Token Management, Location Monitoring, etc.)
  • Main Panel: Card layout displaying different interfaces based on navigation selection
  • Data Integration: Real-time synchronization with PILOT vehicle data and RDM API services
  • Security Layer: JWT token validation and device signature verification

🔌 Data Sources

PILOT Vehicle Data

GET /ax/tree.php?vehs=1&state=1

Loads rental units from PILOT system with hierarchical group structure. Response contains vehicle details including:

  • Vehicle ID (vehid)
  • Unit Name (name)
  • VIN Number (vin)
  • Model & Year (model, year)
  • Device Serial & IMEI (serialnumber, imei)
  • Unit Status (status)

RDM Token API

Complete API integration for token operations:

POST /api/rdm/token/request

Creates new token requests with contract validation

GET /api/rdm/token/status/{serialNumber}

Retrieves current token status and remaining hours

POST /api/rdm/token/generate

Generates JWT tokens with device signature

POST /api/rdm/token/validate

Validates tokens for offline operation

GET /api/rdm/token/list

Lists all tokens with pagination and filtering

POST /api/rdm/token/topup

Adds additional hours to existing tokens

POST /api/rdm/token/renew

Renews token contracts with new periods

GET /api/rdm/token/reports

Generates activity reports and analytics

Contract Data API

GET /api/contracts/{serialNumber}

Retrieves contract information for validation including start/end dates, customer details, and geofence settings.

🔒 Security Features

⚠️ Security Notice: This extension implements enterprise-grade security measures for token management.

JWT Token Structure

  • Header: Algorithm specification and token type
  • Payload: Timer value, contract expiration, device IMEI, system code
  • Signature: Device-specific encrypted signature

Validation Requirements

  • User authentication for all token operations
  • Device ID and signature verification
  • Offline validation capability
  • Cryptographic security for all tokens

Safety Rules

  • Auto-disable only when unit is powered OFF
  • Emergency power button (5-minute activation)
  • Geofence validation before disable commands
  • Breakdown status prevents auto-disable
